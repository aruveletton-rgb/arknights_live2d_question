# 文档目录

本目录只保存项目说明，不保存项目实现。

## 建议阅读顺序

1. [项目概览](PROJECT_OVERVIEW.md)：先了解项目目标、当前状态和边界。
2. [人员职责](TEAM_RESPONSIBILITIES.md)：确认四人三线程分工。
3. [阶段计划](PROJECT_ROADMAP.md)：了解项目从计划到验收的顺序。
4. [系统流程](SYSTEM_FLOW.md)：了解客户端、后端和语音服务怎样协作。
5. [合规边界](COMPLIANCE.md)：确认素材、声音和敏感信息限制。
6. [协作规则](COLLABORATION_WORKFLOW.md)：了解本说明仓库允许提交什么。
7. [PR 提问指南](PULL_REQUEST_GUIDE.md)：按网页步骤向仓库拥有者提问。
8. [资料来源与决定](SOURCES_AND_DECISIONS.md)：查看来源材料和解释依据。

## 文档状态

- `已确认`：仓库拥有者已经明确说明，可作为当前规则。
- `待确认`：仍需要相关人员通过 PR 讨论。
- `未验证`：只有计划或目标，没有实际运行证据。

任何文档都不得把“待确认”或“未验证”改写成“已经完成”。
