# 资料来源与当前决定

## 1. 来源材料

- `Arknights VTuber Project Plan.pdf`：总体目标、四人三线程、阶段、风险和验收计划。
- `GitHub_Collaboration_Workflow_Guide.pdf`：Fork 和 Pull Request 到 `main` 的流程。
- `1号人员计划书.txt`：线程 A 内 1 号职责的补充确认材料。
- `2号人员计划书.txt`：线程 A 内 2 号职责的补充确认材料。

原始文件没有复制进本仓库；这里保存便于非技术参与者阅读的整理结果。

## 2. 人员解释

已确认使用四人、三线程方案：A1/1 号客户端、A2/2 号角色包、B/3 号后端编排、C/4 号语音与部署。

1 号和 2 号计划只确认线程 A 内部职责，不会把整个项目改为两人执行，也不会覆盖 3 号和 4 号。

## 3. 仓库定位

已确认本仓库仅作为说明和提问窗口，不作为实际项目代码仓库。因此：

- 不建立客户端、后端、角色包、部署、脚本或上游源码目录。
- 不保存代码、模型、素材、服务器配置、日志或构建产物。
- 只保存说明、问题、决定和未来实现仓库的公开链接。

## 4. 协作方式

参与者通过短期 `question/` 或 `docs/` 分支发起 PR，目标统一为 `main`。不使用本仓库管理实际开发分支。

## 5. 编码

所有文本文件统一采用无 BOM 的 UTF-8 和 LF 换行。`.editorconfig` 供编辑器读取，`.gitattributes` 负责 Git 文本换行规范。

## 6. 仍待确认

- 各实际实现仓库的地址和负责人权限。
- `Open-LLM-VTuber` 的具体使用方式。
- 项目代码许可证。
- 正式模型、背景和声音的授权。
- 四台服务器的实际可用资源和网络条件。

以上内容确认后，应通过 PR 更新本说明仓库；敏感信息只保存在安全环境中。
