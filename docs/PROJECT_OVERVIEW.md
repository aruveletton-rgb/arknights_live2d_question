# 项目概览

## 1. 项目目标

计划制作一个在 Windows 本地运行的 Live2D 桌宠：用户通过文字或语音交流，远端服务组织角色回复并生成语音，客户端显示字幕、播放音频和切换表情动作。

本项目是非官方同人项目，不隶属于、不代表《明日方舟》、鹰角网络、Yostar 或其他相关官方实体。

## 2. 计划中的组成部分

| 部分 | 作用 | 当前状态 |
| --- | --- | --- |
| Windows 桌宠客户端 | 显示 Live2D、接收输入、播放回复 | 未实现、未验证 |
| 角色包 | 保存角色设定、Prompt、知识和表现规则 | 未实现、未验证 |
| 后端编排 | 接收输入、加载角色、调用 LLM、组织统一回复 | 未实现、未验证 |
| TTS/ASR | 语音合成和语音识别 | 未实现、未验证 |
| 四服务器环境 | 承载网关、语音、备用和健康检查 | 未部署、未验证 |

## 3. 计划中的服务器分工

- VM-1：Gateway + Orchestrator。
- VM-2：TTS Primary。
- VM-3：ASR Gateway。
- VM-4：Fallback + Backup + Healthcheck。

原计划假设每台服务器约为 `2 vCPU / 1 GiB`。这一资源描述尚未通过本仓库登录服务器核实，因此只能作为计划条件。

## 4. 重要限制

- 不在轻量服务器上强行运行大型 LLM、重型 TTS 或重型 ASR。
- 优先使用轻量编排、API 代理、缓存和健康检查。
- 任何服务失败都不应导致桌宠直接崩溃。
- 未经授权的官方素材、角色语音和声优克隆音色不得使用。
- 真实凭据和服务器连接信息不得写进公开说明。

## 5. 本说明仓库的作用

本仓库负责回答“项目是什么、谁负责、怎样协作、有哪些边界”。它不负责保存或运行上述任何组成部分。

未来建立实现仓库后，本仓库可以增加其公开链接和状态说明，但不复制其代码、模型、部署文件或运行日志。
