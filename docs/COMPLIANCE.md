# 素材、声音与敏感信息规则

本项目是非官方同人项目。技术上“能放进仓库”不等于法律或授权上“可以公开使用”。不能确认时，先用自制占位内容并发起提问 PR。

## 1. 可以使用

- 团队成员自行创作并同意用于本项目的内容。
- 有清楚许可条款、允许本项目使用和公开分发的内容。
- 专门为测试制作、不包含第三方受保护内容的占位素材。
- 依法依规使用的第三方代码，并保留其许可证和署名要求。

## 2. 不得提交

- 从官方游戏中拆出的立绘、贴图、Live2D、音频或其他资源。
- 未经授权的官方角色语音或声优克隆音色。
- 来源不明、无法说明作者和许可的模型、背景、声音或 Prompt。
- 密码、API Key、访问令牌、私钥、真实 `.env`、订阅链接或带凭据的日志。
- 为制造官方关系误解而使用的宣传文字、标识或商业素材。

## 3. 每项素材都要登记

使用素材前，在 [`character_pack/ASSET_SOURCE_TABLE.md`](../character_pack/ASSET_SOURCE_TABLE.md) 记录：

- 文件或素材名称；
- 作者或来源；
- 原始链接；
- 许可或授权证明；
- 允许的用途；
- 登记人和日期；
- 当前审核状态。

没有来源记录的素材不得进入公开发布包。

## 4. 声音特别规则

- 默认使用原创音色、明确授权音色或合规的通用 TTS。
- 不把“声音相似”当成获得授权。
- 不克隆官方角色或声优的独特音色。
- TTS 提供方、音色 ID、许可范围和费用限制要写入来源记录。

## 5. 用户数据与日志

- 会话 ID 不使用真实姓名、手机号或邮箱。
- 语音输入只保存完成服务所必需的最短时间。
- 日志不记录 API Key、完整认证头或不必要的完整语音内容。
- 对外分享日志前检查并遮盖服务器地址中的凭据、用户输入和个人信息。

## 6. 发布前检查

- [ ] 项目代码许可证已经确定并放入仓库。
- [ ] 上游项目的许可证和修改说明已经保留。
- [ ] 所有模型、图片、背景、声音和 Prompt 都有来源记录。
- [ ] 没有官方拆包资源或未授权音色。
- [ ] README 明确写明非官方性质。
- [ ] 商业直播、广告、会员或周边用途已经单独重新评估授权风险。

当前许可证与正式素材授权仍未确认，因此仓库目前不具备正式公开发布包的完整条件。
